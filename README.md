# КартриджУчёт

Веб-приложение для учёта расхода картриджей, тонеров, склада и инвентаризации техники в организации. Работает в браузере, не требует установки на рабочих местах.

---

## Быстрый старт (Windows)

### 1. Установить Python
Скачайте Python 3.9+ с https://python.org
При установке отметьте **"Add Python to PATH"**

### 2. Запустить приложение
Дважды щёлкните **`start.bat`**
Скрипт автоматически установит зависимости и запустит сервер.

### 3. Открыть в браузере
Перейдите по адресу: **http://localhost:5000**

### 4. Первый вход
| Логин | Пароль   |
|-------|----------|
| admin | admin123 |

> **Важно:** Смените пароль администратора сразу после первого входа — раздел «Администрирование» → «Пользователи».

## Разделы приложения

### Дэшборд
Главная страница с общей статистикой:
- Текущие затраты на картриджи vs плановые
- Прогноз расходов до конца года
- Сравнение с прошлым годом
- Расход по этажам/подразделениям
- ТОП-7 принтеров по расходу
- Анализ совместимости ПК с Windows 11

### Учёт расхода картриджей
- Таблица всех принтеров с полями ввода количества по месяцам
- Фильтр по этажу
- Автоматический подсчёт итогов и сумм
- История изменений (кто и когда внёс данные)

### Тонеры
- Отдельный учёт тонеров для многофункциональных устройств
- Ввод расхода по месяцам
- Управление списком тонеров (только admin)

### Принтеры
- Реестр всего печатающего оборудования
- Поля: этаж, подразделение, модель, тип картриджа, норма расхода, цена
- Добавление, редактирование, деактивация

### Аналитика
- График план/факт по месяцам
- Сравнение текущего и прошлого года
- Топ подразделений по затратам
- Расход по типам картриджей

### Склад
- Остатки картриджей на складе
- Оповещения о минимальном запасе
- Журнал движения (приход/расход)

### Оборудование
- Инвентаризация ПК, мониторов, телефонов, ИБП
- Поля: ответственный, инвентарный номер, CPU, RAM, HDD, IP, MAC, ОС
- История изменений каждого поля
- Анализ совместимости с Windows 11 (по данным CPU и RAM)

### Администрирование _(только admin)_
- Управление пользователями и ролями
- Создание резервных копий базы данных (хранится до 30 штук)
- Скачивание резервных копий

## Развёртывание

### Локально (один ПК)
```cmd
python app.py
```
Открыть: http://localhost:5000

### В локальной сети (сервер + пользователи)
Подробная инструкция — в файле [NETWORK_SETUP.md](NETWORK_SETUP.md).

Коротко:
1. Скопировать папку на сервер
2. Запустить `start.bat`
3. Открыть порт 5000 в брандмауэре
4. Пользователи открывают `http://[IP-сервера]:5000` в браузере

### Linux / Production
```bash
pip install flask waitress
# С waitress (рекомендуется):
python -c "from waitress import serve; from app import app; serve(app, host='0.0.0.0', port=5000)"
# Или gunicorn:
pip install gunicorn
gunicorn -w 2 -b 0.0.0.0:5000 app:app
```

## Конфигурация

### Секретный ключ сессии
По умолчанию используется встроенный ключ. Для продакшн-сервера задайте свой через переменную окружения:
```cmd
set SECRET_KEY=ваш-длинный-случайный-ключ
python app.py
```
Или скопируйте `.env.example` в `.env` и заполните значения.

### Порт
В конце `app.py` измените `port=5000` на нужный.

### Первоначальные данные
При первом запуске база данных создаётся автоматически и заполняется тестовыми данными (`seed_printers`, `seed_toners` в `app.py`). Замените их на свои перед вводом в эксплуатацию — через интерфейс или отредактировав функции seed в `app.py`.
